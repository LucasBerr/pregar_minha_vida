Arquitetura de Sistemas (Cloud Synced Hybrid)

A solução foi evoluída para uma arquitetura **Real-time Serverless**. O simulador do smartphone ao lado interage dinamicamente com um banco NoSQL em nuvem. Cada alteração de progresso, validação retroativa ou criação de KPIs persiste de forma instantânea e profissional.

Regras de Negócio Fundamentais

  • Chave de Cor Única: Cada KPI cadastrado deve possuir um mapeamento exclusivo de colorId do Google Calendar. O sistema impede a criação de KPIs concorrentes com a mesma cor para evitar sobreposição de contabilidade.
  • Auditoria Humana (Human-in-the-loop): Eventos do calendário fornecem estimativas de tempo planejadas. Apenas tempos ajustados e aprovados na tela de validação diária são de fato somados ao progresso do KPI.
  • Validação Rápida em Lote: Para dias ideais onde a rotina planejada foi seguida perfeitamente, o botão de validação global acelera a rotina diária do usuário.

Estrutura do Banco de Dados em Nuvem (Isolamento por Usuário)

Para obedecer às melhores práticas de privacidade e performance, utilizamos caminhos estruturados que segmentam os dados do usuário atual:

Caminhos de Gravação (Firestore Rule Priority):
Coleção de KPIs:
/artifacts/{appId}/users/{userId}/kpis
Coleção de Eventos:
/artifacts/{appId}/users/{userId}/events
Coleção: KPIs Subcoleção Privada
Campo Tipo Descrição / Exemplo
id String ID gerado automaticamente
name String Ex: "Estudo de Engenharia"
colorId String Mapeamento do Google Calendar (Ex: '7')
goalHours Number Meta semanal contratada

Mecanismo de Sincronização do Calendário

O motor do aplicativo conecta-se à API do Google para buscar as atividades de forma assíncrona. Os registros retornados são comparados com a base de dados em nuvem para filtrar o que já foi validado pelo usuário.

Endpoint REST Utilizado:
